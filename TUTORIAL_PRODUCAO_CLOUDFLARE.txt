TUTORIAL: SUBIR O PAINEL AUTO DEPLOY CLOUDFLARE EM PRODUCAO
============================================================

Este tutorial documenta a configuracao atual do painel em producao.

Projeto local:
C:\Users\NATHAN-PC\Desktop\AutoDeployCloudFlare-main

URL atual de producao:
https://auto-deploy-cloudflare.nathan-aguiar-masson.workers.dev


1. Entrar na pasta do projeto
-----------------------------

cd C:\Users\NATHAN-PC\Desktop\AutoDeployCloudFlare-main


2. Fazer login no Wrangler
--------------------------

.\node_modules\.bin\wrangler.cmd login

O navegador abre para login na Cloudflare.

Se o Wrangler perguntar sobre instalar Cloudflare skills para agentes de IA,
pode responder:

n


3. Criar ou confirmar o banco D1 remoto
---------------------------------------

Criar:

.\node_modules\.bin\wrangler.cmd d1 create auto-deploy-cloudflare-db

Banco atual:

database_name: auto-deploy-cloudflare-db
database_id: 33397f17-8417-49eb-a44e-c426410f7f45

No wrangler.jsonc, o binding precisa ser APP_DB:

"d1_databases": [
  {
    "binding": "APP_DB",
    "database_name": "auto-deploy-cloudflare-db",
    "database_id": "33397f17-8417-49eb-a44e-c426410f7f45",
    "migrations_dir": "./migrations"
  }
]


4. Aplicar migrations no D1 remoto
----------------------------------

.\node_modules\.bin\wrangler.cmd d1 migrations apply auto-deploy-cloudflare-db --remote

Migrations atuais:
- 0001_initial.sql: tabelas iniciais
- 0002_clients.sql: clientes e settings Cloudflare por cliente
- 0003_users.sql: usuarios, roles e login multiusuario


5. Criar ou confirmar bucket R2 remoto
--------------------------------------

Criar:

.\node_modules\.bin\wrangler.cmd r2 bucket create auto-deploy-cloudflare-artifacts

No wrangler.jsonc, o binding principal precisa ser APP_BUCKET:

"r2_buckets": [
  {
    "binding": "APP_BUCKET",
    "bucket_name": "auto-deploy-cloudflare-artifacts"
  }
]


6. Configurar secrets do Worker
-------------------------------

Secrets necessarios:

APP_ADMIN_SECRET
TOKEN_ENCRYPTION_KEY
GITHUB_ACTIONS_TOKEN
RUNNER_SHARED_SECRET

Comandos:

.\node_modules\.bin\wrangler.cmd secret put APP_ADMIN_SECRET
.\node_modules\.bin\wrangler.cmd secret put TOKEN_ENCRYPTION_KEY
.\node_modules\.bin\wrangler.cmd secret put GITHUB_ACTIONS_TOKEN
.\node_modules\.bin\wrangler.cmd secret put RUNNER_SHARED_SECRET

Confirmar nomes dos secrets:

.\node_modules\.bin\wrangler.cmd secret list

Importante:
- APP_ADMIN_SECRET e usado para criar o primeiro admin quando a tabela users esta vazia.
- TOKEN_ENCRYPTION_KEY criptografa tokens Cloudflare dos clientes no D1.
- GITHUB_ACTIONS_TOKEN dispara a workflow GitHub Actions.
- RUNNER_SHARED_SECRET protege a comunicacao entre GitHub Actions e o painel.

Nao troque TOKEN_ENCRYPTION_KEY depois que tokens Cloudflare ja foram salvos,
a menos que voce va cadastrar os tokens novamente.


7. Configurar secret no GitHub Actions
--------------------------------------

No repositorio:

nathanmasson1/AutoDeployCloudFlareGitHubActions

Abra:

Settings > Secrets and variables > Actions

Crie ou atualize:

RUNNER_SHARED_SECRET

O valor deve ser exatamente igual ao RUNNER_SHARED_SECRET salvo no Worker.


8. Criar GitHub token do administrador
--------------------------------------

Crie um Fine-grained personal access token da conta administradora.

Permissoes recomendadas:
- Repository access: apenas o repositorio do painel
- Actions: Read and write
- Contents: Read-only

Salve no Worker:

.\node_modules\.bin\wrangler.cmd secret put GITHUB_ACTIONS_TOKEN

Nunca cole esse token em chat, print, issue ou documento publico.
Se vazar, revogue no GitHub e gere outro.


9. Configurar variaveis nao secretas
------------------------------------

No wrangler.jsonc, em vars:

SESSION_TTL_SECONDS = 604800
APP_BASE_URL = https://auto-deploy-cloudflare.nathan-aguiar-masson.workers.dev
GITHUB_ACTIONS_OWNER = nathanmasson1
GITHUB_ACTIONS_REPO = AutoDeployCloudFlareGitHubActions
GITHUB_ACTIONS_WORKFLOW = direct-deploy.yml
GITHUB_ACTIONS_REF = main


10. Fazer deploy do painel
--------------------------

npm run build
.\node_modules\.bin\wrangler.cmd deploy

Ou:

npm run deploy


11. Primeiro login admin
------------------------

Se a tabela users estiver vazia, o primeiro login cria o admin inicial.

Use:

Email: admin@local
Senha: valor atual do APP_ADMIN_SECRET

Depois de entrar:
1. Va em Credenciais.
2. Na area Usuarios, edite ou crie o admin com seu email real.
3. Crie usuarios clientes vinculados aos clientes corretos.


12. Configurar clientes e credenciais Cloudflare
------------------------------------------------

Dentro do painel:

1. Entrar em Credenciais.
2. Criar ou selecionar um cliente.
3. Clicar em Criar token pre-preenchido.
4. Criar o token na Cloudflare do cliente.
5. Copiar o token gerado.
6. Colar o token no painel.
7. O painel busca Account ID e Account name automaticamente.
8. Confirmar plano pago quando necessario.
9. Salvar.

Cada cliente tem sua propria Cloudflare API Token.
Usuarios cliente veem apenas o cliente vinculado.


13. Permissoes esperadas no token Cloudflare do cliente
-------------------------------------------------------

O token Cloudflare criado para cada cliente deve ter permissoes semelhantes a:

Account > D1 > Edit
Account > Workers R2 Storage > Edit
Account > Workers KV Storage > Edit
Account > Workers Scripts > Edit
Account > Account Settings > Edit
Zone > Zone > Read
Zone > Zone > Edit
Zone > DNS > Edit
Zone > Workers Routes > Edit

Tambem configurar:

Account Resources: Include > conta Cloudflare do cliente
Zone Resources: Include > All zones ou os dominios usados

Workers Builds Configuration nao e necessario neste modo.
Workers Builds API token nao e necessario.
GitHub App da Cloudflare nao e necessario.


14. Dados locais e producao
---------------------------

Os dados de localhost ficam no D1 local do Wrangler.
O D1 remoto de producao e separado.

Dados locais nao aparecem automaticamente em producao.

16. Erros comuns
----------------

GITHUB_ACTIONS_TOKEN nao configurado para disparar GitHub Actions.
- Em producao: secret faltando/vazio ou Worker nao redeployado.
- Corrigir:
  .\node_modules\.bin\wrangler.cmd secret put GITHUB_ACTIONS_TOKEN
  .\node_modules\.bin\wrangler.cmd deploy
- Em localhost: crie .dev.vars e reinicie wrangler dev.

Runner nao autorizado.
- RUNNER_SHARED_SECRET no GitHub e no Worker estao diferentes.
- Atualize os dois com o mesmo valor.

Pbkdf2 failed: iteration counts above 100000 are not supported.
- Codigo antigo publicado.
- Rode npm run build e wrangler deploy.

Nao consigo entrar no admin.
- Se users estiver vazio, use admin@local + APP_ADMIN_SECRET.
- Para conferir usuarios:
  .\node_modules\.bin\wrangler.cmd d1 execute auto-deploy-cloudflare-db --remote --command "SELECT email, role, active FROM users;"

Token Cloudflare descriptografia falhou.
- TOKEN_ENCRYPTION_KEY mudou depois que os tokens foram salvos.
- Use a chave antiga ou cadastre novamente os tokens Cloudflare.
