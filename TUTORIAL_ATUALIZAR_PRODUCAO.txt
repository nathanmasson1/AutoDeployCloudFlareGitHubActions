TUTORIAL: ATUALIZAR PRODUCAO DEPOIS DE EDITAR O CODIGO LOCAL
=============================================================

Use este guia quando voce alterar o codigo no localhost e quiser mandar a nova
versao para producao na Cloudflare.

Projeto:
C:\Users\NATHAN-PC\Desktop\AutoDeployCloudFlare-main


1. Entrar na pasta do projeto
-----------------------------

cd C:\Users\NATHAN-PC\Desktop\AutoDeployCloudFlare-main


2. Editar e testar localmente
-----------------------------

Terminal 1:

.\node_modules\.bin\wrangler.cmd dev --ip 127.0.0.1 --port 8787

Terminal 2:

npm run dev

Abra:

http://127.0.0.1:5173/

Importante:
Alterar codigo local NAO altera producao automaticamente.
Producao so muda depois de rodar deploy.


3. Secrets em localhost
-----------------------

O Worker local nao usa automaticamente os secrets remotos da Cloudflare.
Se for testar login/deploy em localhost, crie ou atualize o arquivo:

.dev.vars

Exemplo:

APP_ADMIN_SECRET=SUA_SENHA_ADMIN_LOCAL
TOKEN_ENCRYPTION_KEY=SUA_CHAVE_DE_16_OU_MAIS_CARACTERES
GITHUB_ACTIONS_TOKEN=COLE_SEU_GITHUB_PAT_AQUI
RUNNER_SHARED_SECRET=COLE_O_MESMO_RUNNER_SHARED_SECRET_AQUI
APP_BASE_URL=https://auto-deploy-cloudflare.nathan-aguiar-masson.workers.dev
GITHUB_ACTIONS_OWNER=nathanmasson1
GITHUB_ACTIONS_REPO=AutoDeployCloudFlareGitHubActions
GITHUB_ACTIONS_WORKFLOW=direct-deploy.yml
GITHUB_ACTIONS_REF=main
SESSION_TTL_SECONDS=604800

Depois reinicie o wrangler dev.


4. Validar antes de publicar
----------------------------

Antes de mandar para producao, rode:

npm run build

Esse comando faz:

npm run build:client
tsc --noEmit

Se der erro, corrija antes de publicar.


5. Publicar a nova versao
-------------------------

Depois que o build passar:

npm run deploy

Esse comando faz:

npm run build
wrangler deploy

O deploy atualiza:
- Worker em src/worker
- Frontend React/Vite compilado para dist/client
- Codigo compartilhado em src/shared
- Variaveis nao secretas do wrangler.jsonc

O deploy NAO envia:
- Dados locais do D1
- Arquivo .dev.vars
- Valores dos secrets
- Mudancas de banco ainda sem migration remota


6. Quando precisa rodar migration
---------------------------------

Rode migration quando criar ou alterar arquivos em:

migrations

Comando:

.\node_modules\.bin\wrangler.cmd d1 migrations apply auto-deploy-cloudflare-db --remote

Migrations importantes atuais:
- 0001_initial.sql: tabelas iniciais
- 0002_clients.sql: clientes e settings por cliente
- 0003_users.sql: usuarios, roles e login multiusuario

Fluxo quando alterou banco:

npm run build
.\node_modules\.bin\wrangler.cmd d1 migrations apply auto-deploy-cloudflare-db --remote
npm run deploy


7. Quando precisa mexer em secrets
----------------------------------

Voce precisa rodar secret put quando:
- Trocar senha inicial/admin
- Trocar chave de criptografia
- Trocar GitHub PAT
- Trocar segredo do runner
- Criar um novo secret usado pelo codigo

Comandos:

.\node_modules\.bin\wrangler.cmd secret put APP_ADMIN_SECRET
.\node_modules\.bin\wrangler.cmd secret put TOKEN_ENCRYPTION_KEY
.\node_modules\.bin\wrangler.cmd secret put GITHUB_ACTIONS_TOKEN
.\node_modules\.bin\wrangler.cmd secret put RUNNER_SHARED_SECRET

Depois de mudar secrets, rode:

.\node_modules\.bin\wrangler.cmd deploy

Importante sobre TOKEN_ENCRYPTION_KEY:
Nao troque essa chave depois que ja existem tokens salvos no painel, a menos que
voce esteja pronto para cadastrar novamente os tokens Cloudflare dos clientes.

Importante sobre GITHUB_ACTIONS_TOKEN:
Se o token GitHub foi colado em chat, print, issue ou documento publico,
revogue no GitHub e crie outro.


8. Login admin e multiusuario
-----------------------------

Se a tabela users estiver vazia, o primeiro login cria o admin inicial.

Use:

Email: admin@local
Senha: valor atual do APP_ADMIN_SECRET

Depois que entrar, va em Credenciais > Usuarios e crie:
- Admins adicionais, se quiser
- Usuarios clientes vinculados a um cliente

Usuarios cliente veem apenas:
- O proprio cliente
- Sites do proprio cliente
- Jobs/logs do proprio cliente
- Credenciais Cloudflare do proprio cliente


9. Quando precisa mexer em wrangler.jsonc
-----------------------------------------

Altere wrangler.jsonc quando mudar:
- Nome do Worker
- Bindings D1/R2/assets
- Database ID
- Bucket R2
- Variaveis nao secretas em vars
- Compatibility date/flags

Bindings que o codigo espera:

D1:
binding = APP_DB

R2:
binding = APP_BUCKET

Assets:
binding = ASSETS

Nao renomeie APP_DB, APP_BUCKET ou ASSETS sem atualizar o codigo.


10. Verificar se publicou certo
-------------------------------

Depois do deploy, abra:

https://auto-deploy-cloudflare.nathan-aguiar-masson.workers.dev

Confira:
- Login funciona
- Credenciais abrem
- Usuarios aparecem para admin
- Templates aparecem
- Projetos publicados aparecem
- Criar site funciona

Se a producao nao mudar:
1. Recarregue com Ctrl + F5.
2. Confira se npm run deploy terminou com sucesso.
3. Confira se voce esta na URL de producao correta.
4. Rode npm run deploy novamente.


11. Erros comuns
----------------

Pbkdf2 failed: iteration counts above 100000 are not supported
- Codigo antigo publicado.
- Rode npm run build e npm run deploy.

GITHUB_ACTIONS_TOKEN nao configurado
- Em producao: rode secret put GITHUB_ACTIONS_TOKEN e depois deploy.
- Em localhost: crie .dev.vars e reinicie wrangler dev.

Runner nao autorizado
- RUNNER_SHARED_SECRET do Worker e do GitHub estao diferentes.
- Atualize os dois lugares com o mesmo valor.

Login nao funciona e nao sei o email
- Se users estiver vazio, use admin@local.
- Se ja existir usuario, consulte emails no D1 remoto:
  .\node_modules\.bin\wrangler.cmd d1 execute auto-deploy-cloudflare-db --remote --command "SELECT email, role, active FROM users;"
